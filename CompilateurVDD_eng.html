Using the compiler :


Compilation :    javac Main.java
or                           javac -cp . Main.java                


Usage :         java Main [args]
or                          java -cp . Main [args]   

        arg1 : file.xml (name of the file to compile)
        arg2 : file type : "plus" or "time" (also accepted "p" "t" "+")
        arg3 : heuristic number, with 1:MCF, 2:Band-Width, 3:MCS-Inv, 4:Force, 5:MCS+1
        arg4 : final form : "AADD", "SLDDp", "SLDDt"
        (arg5 : save_file.dot)
        (arg6 : Keep useless nodes : "NoSkip")


Using case :

   java Main small + 3 AADD smallAADD NoSkip
   
   
    This commande compile the small.xml file (additive type file) with MCS-Inv heuristic. this file is necessarily compiled in SLDD+, and then transformed in AADD and saved in smallAADD.dot and smallAADD.xml. Useless nodes (i.e. nodes with no constraints) are kept.
 

More details on arguments :

    Compilation :
        arg1 : this file must be in the same file than this program (or you must enter the full adress)
        arg2 : say if constraints are additives (ex : WCSP) or multipicatives (ex : bayesian network).
        arg3 : only an integer is accepted.
        arg4 : this fonction will automaticly compilled into SLDD+ or SLDDt, and then translated into this format.
        arg5 : ras
        arg6 : Useless nodes are kept. Every variable will be seen regardless to the way.

File format :

   input xml files : see
        -Olivier Roussel and Christophe Lecoutre. XML Representation of Constraint Networks : Format XCSP 2.1.Technical report, CoRR abs/0902.2362, feb 2009.
        -Fabio Gagliardi Cozman. JavaBayes Version 0.347, Bayesian Networks in Java, User Manual. Technical report, dec 2002. Benchmarks at http ://sites.poli.usp.br/pmr/ltd/Software/javabayes/Home/node3.html.

    output dot files :
        .dot : This format use Graphviz dot formalism, so diagram can be visualised with this tool.
        .xml : xml representation of the diagram
